TECH STACK
----------
- Use Next.js with the App Router
- Tailwind CSS for styling
- No external UI libraries needed
- Keep dependencies minimal
- The site should be fully static-export compatible (next export)
- Use next/font for typography

---

DESIGN DIRECTION
----------------
- Clean, minimal, editorial — think a well-designed literary magazine or personal journal, not a corporate portfolio
- Color palette: off-white or light cream background, near-black text, one subtle accent color (a muted warm tone like dusty amber or slate blue — choose whichever feels right for the content)
- Typography: a readable serif for body text (like Playfair Display or Lora via Google Fonts), a clean sans-serif for navigation and labels (like Inter or DM Sans)
- Generous whitespace and line-height — this is a reading site, readability is the top priority
- Subtle hover states, no animations that distract from the writing
- Fully responsive — mobile first
- No hero images or stock photos — the writing is the visual focus
- Each page should feel like opening a well-formatted piece of writing, not a slideshow

READING THE DOCX FILES
-----------------------
Use the mammoth npm package to extract text from .docx files at build time (or in a server component). This preserves the paragraph structure cleanly.

Install: npm install mammoth

For each .docx file, read it in a Next.js server component or generateStaticParams function, extract the text using mammoth.extractRawText(), then parse the paragraphs and render them.

For the PDF files in the Destination folder, use pdf-parse to extract the text:
Install: npm install pdf-parse

Render all body text as proper <p> tags. Detect numbered list items (lines starting with a number followed by a period) and render them as <ol> with <li> elements. Detect bold headings within list items by checking if the item heading is on its own line before the paragraph.

TYPOGRAPHY AND READING EXPERIENCE
----------------------------------
- Body text: 18-19px, line-height 1.75, max-width 680px
- Essay titles: large, serif, prominent
- Section labels (Essay, Sidebar): small caps or uppercase, letter-spaced, muted color — these are labels not headings
- Transformation list item numbers: styled clearly, bold
- Transformation list item headings (the bold titles for each numbered point): render in bold, slightly larger than body
- Paragraph spacing: generous — 1.2-1.5em between paragraphs
- On the Guide page, the sidebar column should have slightly smaller body text (16px) to distinguish it from the main essay
